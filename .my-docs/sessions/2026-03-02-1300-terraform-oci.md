- compute.tf를 for_each 방식에서 단일 인스턴스 방식으로 변경, outputs.tf도 동기화

### 2. 앱 배포를 위한 코드 수정
- `backend/main.py`: CORS_ORIGINS 환경변수로 동적 오리진 추가 지원
- `frontend-ts/Dockerfile`: NEXT_PUBLIC_API_URL을 ARG로 변경 (빌드 시 주입)
- `docker-compose.yml`: frontend build.args 추가, backend 소스 마운트 제거, CORS_ORIGINS 환경변수 추가
- `.env.prd`: 강력한 비밀번호 생성, NEXT_PUBLIC_API_URL/CORS_ORIGINS 추가
- `deploy.sh`: rsync 기반 원격 배포 스크립트 신규 작성

### 3. 배포 실행 및 네트워크 이슈 해결
- rsync로 서버에 코드 전송 후 docker compose up --build 실행
- Docker 빌드 중 DNS 실패 → iptables가 Docker 브릿지 트래픽 차단 중이었음 → docker0 인터페이스 ACCEPT 룰 추가로 해결
- 외부 접속 불가 → OCI Security List에 포트 3000 누락 → network.tf에 추가 후 terraform apply
- DOCKER iptables 체인의 DROP 룰 → iptables 전체 리셋 + Docker 재시작 + 클린 룰 재적용으로 해결

### 4. cloud-init.yaml Docker 호환 iptables로 업데이트
- `iptables -F` → `iptables -F INPUT` (INPUT 체인만 플러시, Docker FORWARD/NAT 보존)
- docker0 인터페이스 ACCEPT 룰 추가
- 포트 3000 ACCEPT 룰 추가

- **OCI Free Tier ARM 용량 부족**: 춘천 리전에서 ARM 인스턴스 확보 실패가 빈번. PAYG 전환으로 우선순위 상승하여 즉시 확보 성공
